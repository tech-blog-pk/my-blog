CSS Custom Properties: Stop Hard-Coding Your Styles

MK
Malik Khan · May 5, 2026 7 min read

If you're still writing color: #2d6a4f in fifteen different places across your stylesheet, this article is for you. CSS Custom Properties — often called CSS variables — are one of the most useful features in modern CSS, and they're underused by a lot of developers who learned CSS before they existed.

What Are CSS Custom Properties?

CSS Custom Properties let you define a value once and reuse it across your entire stylesheet. If you need to change it, you change it in one place — and it updates everywhere.

/* Define your variables on :root */
:root {
  --color-brand: #2d6a4f;
  --color-bg: #f7f6f2;
  --font-mono: 'DM Mono', monospace;
  --spacing-lg: 2rem;
}

/* Use them anywhere */
.button {
  background: var(--color-brand);
  font-family: var(--font-mono);
  padding: var(--spacing-lg);
}

The syntax is two dashes before the name (--name) for defining, and var(--name) for using. That's the whole syntax.

Why Not Just Use Sass Variables?

This is a fair question. Sass variables ($color: red;) have existed for years. But CSS Custom Properties have one massive advantage: they're live in the browser.

Sass variables are compiled away before the browser sees them. CSS Custom Properties exist at runtime, which means:

  • You can change them with JavaScript
  • They respect the cascade and can be scoped to specific elements
  • They work with media queries and pseudo-classes
  • No build step required

Scoping Variables to Elements

You're not limited to :root. You can define a variable on any element, and it will only apply within that element's subtree.

/* Global default */
:root {
  --card-bg: #ffffff;
}

/* Override for dark sections */
.dark-section {
  --card-bg: #1a1a18;
}

.card {
  background: var(--card-bg); /* automatically picks up local value */
}
This scoping behaviour is what makes CSS Custom Properties more powerful than preprocessor variables for theming.

Dark Mode in 10 Lines

CSS variables make implementing dark mode surprisingly clean. Define your light theme on :root, then override the variables inside a media query.

:root {
  --bg: #ffffff;
  --text: #1a1a18;
  --border: #e0e0e0;
}

@media (prefers-color-scheme: dark) {
  :root {
    --bg: #1a1a18;
    --text: #f7f6f2;
    --border: #333330;
  }
}

body {
  background: var(--bg);
  color: var(--text);
}

Every element that uses these variables will automatically switch when the user's OS is in dark mode. No JavaScript required.

Changing Variables with JavaScript

Because CSS Custom Properties are live, you can read and write them from JavaScript. This is great for things like user-controlled themes or animation-driven UI.

// Read a variable
const brand = getComputedStyle(document.documentElement)
  .getPropertyValue('--color-brand');

// Write a variable
document.documentElement.style
  .setProperty('--color-brand', '#52b788');

// All elements using --color-brand update instantly!

Fallback Values

The var() function accepts a second argument as a fallback, used when the variable isn't defined.

.card {
  /* Use --card-radius if defined, otherwise 8px */
  border-radius: var(--card-radius, 8px);
}

This is useful for building component libraries where consumers can optionally override your defaults.

Practical: Build a Design Token System

The best use of CSS Custom Properties is as a design token system — a central source of truth for every visual decision in your project.

:root {
  /* Colors */
  --color-primary: #2d6a4f;
  --color-primary-light: #d8f3dc;
  --color-neutral-100: #f7f6f2;
  --color-neutral-900: #1a1a18;

  /* Typography */
  --font-sans: 'DM Sans', sans-serif;
  --font-mono: 'DM Mono', monospace;
  --font-serif: 'Fraunces', serif;
  --text-sm: 0.85rem;
  --text-base: 1rem;
  --text-lg: 1.2rem;

  /* Spacing */
  --space-1: 0.25rem;
  --space-2: 0.5rem;
  --space-4: 1rem;
  --space-8: 2rem;

  /* Radius */
  --radius-sm: 6px;
  --radius-md: 10px;
  --radius-full: 100px;
}

With a system like this, your entire design becomes consistent by default. Changing the primary brand colour means updating one line.

What to Watch Out For

  • Invalid values: If a variable contains an invalid value for a property, the browser uses the property's inherited or initial value — it doesn't throw an error, which can make bugs hard to spot.
  • No calc() short-hand: You can use variables inside calc(), but the variable must include the unit: var(--size) → 16px, not 16.
  • Browser support: Full support in all modern browsers. IE11 does not support them — but if you're still targeting IE11, you have bigger problems.

Summary

CSS Custom Properties are not just a convenience feature. Used well, they give you a maintainable design system, effortless theming, and live JavaScript integration — all in native CSS. Start by moving your repeated colours and font stacks to :root variables. You'll immediately feel the difference.
