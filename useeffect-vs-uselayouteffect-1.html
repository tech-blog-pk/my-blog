useEffect vs useLayoutEffect: When Each One Actually Matters

Most React developers use useEffect for everything. But there's a subtle — and critical — difference between these two hooks that can cause visual bugs in your UI if you pick the wrong one.

AK
Ali Khan · Apr 29, 2026 · 6 min read
ReactHooksJavaScript

React has two hooks that look almost identical on the surface: useEffect and useLayoutEffect. Both accept a callback and a dependency array. Both run after a component renders. So what's the difference — and does it actually matter?

Yes, it matters a lot. Getting this wrong can cause visible flickering in your UI. Let's break it down clearly.

1. How React Renders — A Quick Recap

Before we compare the hooks, we need to understand the browser's rendering pipeline. When React updates your component, here's what happens in order:

Step 1: React calculates the changes (virtual DOM diffing)
Step 2: React updates the actual DOM
Step 3: The browser paints — displays the changes on screen
Step 4: Your effects run

The key question is: does your effect run before or after the browser paints? That's exactly what separates useEffect from useLayoutEffect.

2. useEffect — Runs After Paint

useEffect runs after the browser has already painted the screen. This means the user sees the updated UI first, and then your effect runs in the background.

React
import { useState, useEffect } from 'react';

function UserProfile({ userId }) {
  const [user, setUser] = useState(null);

  useEffect(() => {
    // Runs AFTER the browser paints
    // Perfect for: API calls, subscriptions, timers
    fetchUser(userId).then(data => setUser(data));
  }, [userId]);

  return <div>{user ? user.name : 'Loading...'}</div>;
}

Best for: API calls, data fetching, event listeners, subscriptions, logging, and anything that doesn't need to happen before the user sees the screen.

3. useLayoutEffect — Runs Before Paint

useLayoutEffect runs synchronously after React updates the DOM, but before the browser paints. This means you can measure or modify the DOM and the user will never see the intermediate state.

React
import { useState, useRef, useLayoutEffect } from 'react';

function Tooltip({ text }) {
  const ref = useRef(null);
  const [position, setPosition] = useState({ top: 0, left: 0 });

  useLayoutEffect(() => {
    // Runs BEFORE the browser paints
    // Measure the DOM element and set position
    const rect = ref.current.getBoundingClientRect();
    setPosition({ top: rect.bottom, left: rect.left });
  }, []);

  return (
    <div ref={ref} style={{ top: position.top, left: position.left }}>
      {text}
    </div>
  );
}

Warning: useLayoutEffect blocks the browser from painting until it finishes. Heavy work inside it will make your app feel slow. Keep it fast and focused.

4. Side-by-Side Comparison

useEffect

Runs after browser paint. Non-blocking. Best for async work like API calls, subscriptions, and timers.

useLayoutEffect

Runs before browser paint. Blocking. Best for DOM measurements and preventing visual flicker.

5. The Flicker Problem — A Real Example

Here's a practical case where using the wrong hook causes a visible bug. Imagine a component that needs to measure its own width and adjust its layout based on that measurement.

❌ With useEffect — causes flicker

React — Causes flicker
function ResizableBox() {
  const ref = useRef(null);
  const [width, setWidth] = useState(0);

  useEffect(() => {
    // Browser paints FIRST with width = 0
    // Then this runs — user sees a flicker!
    setWidth(ref.current.offsetWidth);
  }, []);

  return <div ref={ref} style={{ fontSize: width > 400 ? '18px' : '14px' }}>
    Content here
  </div>;
}

✅ With useLayoutEffect — no flicker

React — No flicker
function ResizableBox() {
  const ref = useRef(null);
  const [width, setWidth] = useState(0);

  useLayoutEffect(() => {
    // Runs BEFORE paint — user never sees width = 0
    setWidth(ref.current.offsetWidth);
  }, []);

  return <div ref={ref} style={{ fontSize: width > 400 ? '18px' : '14px' }}>
    Content here
  </div>;
}

6. The Simple Decision Rule

Here's how to decide which hook to use every time:

✅ Use useEffect — for 95% of cases. API calls, subscriptions, logging, timers.
✅ Use useLayoutEffect — only when you need to read or write the DOM before the user sees it. Tooltips, animations, measuring element size.

Golden Rule: Start with useEffect. Only switch to useLayoutEffect if you see visual flickering that needs to be fixed.

Summary

The difference is simple but important. useEffect is your default — it runs after the browser paints and is perfect for async work. useLayoutEffect is a specialized tool — it runs before the browser paints and is used to fix visual flickering when you need to measure or modify the DOM immediately.

When in doubt, use useEffect. Your users won't notice the difference in most cases, and your app will stay performant.
